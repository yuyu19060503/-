# 📋 开发计划

> 版本：v1.0 | 最后更新：2026-06-01

---

## 总体原则

- **小步迭代**：每个阶段产出可运行的产物，不堆砌代码
- **稳定优先**：每阶段结束确保无报错、基础功能可用
- **用户可见**：每阶段结束用户都能看到新东西

---

## 阶段概览

| 阶段 | 名称 | 预计产出 |
|------|------|----------|
| Phase 1 | 项目初始化 | 能跑起来的空壳 App |
| Phase 2 | 食材选择 UI | 可交互的食材网格页面 |
| Phase 3 | AI 推荐 | 能调 API 出推荐结果 |
| Phase 4 | 菜品详情 | 制作步骤展示 |
| Phase 5 | 辅助功能 | 收藏、历史、冰箱、筛选 |
| Phase 6 | 打磨上线 | UI 细节、动画、错误处理 |

---

## Phase 1：项目初始化

**目标**：搭建 Expo 项目，确保 iOS / Android 能跑起来

- [ ] 使用 `create-expo-app` 初始化项目
- [ ] 配置 TypeScript
- [ ] 配置主题常量（颜色、字体）
- [ ] 搭建底部 Tab 导航骨架（4 个 Tab，占位页面）
- [ ] 验证 iOS 模拟器运行
- [ ] 验证 Android 模拟器运行

**完成标准**：App 在模拟器上显示底部 4 个 Tab，点击可切换

---

## Phase 2：食材选择 UI

**目标**：首页可展示食材网格、选中/取消交互

- [ ] 编写食材数据文件（80–100 种食材，分类、emoji、名称）
- [ ] 实现分类标签栏组件
- [ ] 实现食材卡片组件（选中/未选中两种状态）
- [ ] 实现食材网格（按分类过滤）
- [ ] 实现已选食材底部横条
- [ ] 响应式布局适配

**完成标准**：用户可点击食材，选中效果正常，分类切换正常

---

## Phase 3：AI 推荐

**目标**：点击找菜能调 AI 并展示推荐结果

- [ ] 封装 Claude API 调用（Prompt 工程 + 响应 JSON 解析）
- [ ] 实现推荐结果页 UI
- [ ] 实现加载状态（loading 动画）
- [ ] 实现错误处理（网络异常提示）
- [ ] 「换一个」逻辑（排除已推荐）
- [ ] 缺料提示展示

**完成标准**：选中食材 → 找菜 → AI 返回推荐 → 展示菜品卡片

---

## Phase 4：菜品详情

**目标**：展示完整制作步骤

- [ ] 菜品详情页 UI（食材匹配、步骤列表）
- [ ] 步骤化排版
- [ ] 收藏按钮交互
- [ ] 自动记录历史

**完成标准**：推荐结果 → 就这个 → 查看步骤 → 可收藏

---

## Phase 5：辅助功能

**目标**：完整的产品闭环

- [ ] 收藏页面（列表展示、取消收藏）
- [ ] 历史记录页面（时间倒序、清除）
- [ ] 冰箱页面（增删食材、持久化）
- [ ] 筛选偏好弹窗（菜系、难度、时间）
- [ ] 筛选参数接入 AI 请求

**完成标准**：所有辅助功能可用，数据持久化正常

---

## Phase 6：打磨上线

**目标**：高质量交付

- [ ] UI 细节走查（间距、颜色、空状态）
- [ ] 动画微交互实现
- [ ] 无障碍适配（大字体支持）
- [ ] 性能优化（食材列表 FlatList 虚拟化）
- [ ] 构建测试（Expo EAS Build）
- [ ] 生成 APK / IPA 安装包

**完成标准**：可安装使用，无明显 bug
